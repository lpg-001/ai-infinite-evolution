# Father Skill - 主技能管理器

## 系统定位

Father Skill是AI无限进化系统的**核心管理器**和**协调中心**，在整个技能生态系统中扮演着至关重要的角色。它负责整个技能系统的生命周期管理、更新协调和优先级控制，是连接san技能和order技能的桥梁，确保系统能够持续学习和进化。

**核心价值定位**：
- **统一管理**：作为技能系统的中央协调器，统一管理所有技能模块
- **持续进化**：通过自动化的更新机制，确保系统能够从每次对话中学习和改进
- **优先级保障**：维护明确的优先级规则，确保重要规则优先应用
- **无缝集成**：与Cursor IDE深度集成，实现自动加载和执行

## 核心职责

### 1. 技能生命周期管理

Father Skill管理整个技能系统的完整生命周期，从对话开始到结束的每一个阶段都有明确的职责：

**对话前阶段**：
- **技能加载**：按优先级顺序加载所有技能模块
- **上下文初始化**：建立对话上下文，应用已学习的规则
- **钩子预检测**：检测用户输入中是否包含钩子命令
- **环境准备**：准备执行环境和必要的资源

**对话中阶段**：
- **实时监控**：监控技能执行情况，确保规则正确应用
- **动态协调**：协调不同技能模块之间的交互
- **钩子执行**：如果检测到钩子，协调执行对应的技能
- **响应生成**：根据规则生成高质量的响应

**对话后阶段**：
- **内容分析**：深入分析对话内容，提取可学习的模式
- **更新判断**：判断哪些技能需要更新
- **更新执行**：执行更新操作，优化技能内容
- **日志记录**：记录更新日志，追踪系统进化轨迹

### 2. 优先级管理

维护明确的技能加载和应用优先级，确保系统行为的一致性和可预测性：

```
优先级层次（从高到低）：
1. san/SKILL.md      ← 最高优先级，用户输入语言规则和AI思考规则
   └─> 这些规则直接决定AI如何理解和响应
   
2. father/SKILL.md   ← 主技能管理器（当前文件）
   └─> 管理规则和协调机制，但不覆盖san规则
   
3. order/SKILL.md    ← 其他技能（通过钩子触发）
   └─> 特定功能实现，只在钩子触发时应用
```

**优先级规则详解**：

- **覆盖原则**：高优先级规则完全覆盖低优先级规则，不允许反向覆盖
- **san规则优先**：san规则始终优先应用，father规则不能修改或覆盖san规则
- **条件应用**：order规则只在钩子触发时应用，不会影响正常对话流程
- **冲突解决**：当不同优先级规则出现冲突时，严格按照优先级顺序，以高优先级为准
- **一致性保证**：优先级规则确保系统行为的一致性，避免规则冲突导致的混乱

### 3. 更新协调

协调各技能模块的更新流程，确保系统能够持续学习和进化：

**更新类型**：

- **强制更新**：san/SKILL.md每次对话后必须更新，这是系统进化的核心机制
  - 即使没有新内容，也要进行优化性更新
  - 必须明确说明更新了什么内容
  - 更新内容要详细、准确、可追溯

- **可选更新**：order/SKILL.md仅在需要时更新
  - 只有当对话涉及相关技能时才更新
  - 新增技能时必须添加对应的钩子定义
  - 更新时需要说明添加或修改了哪些钩子和技能

**更新验证机制**：

- **格式验证**：确保更新内容符合markdown格式规范
- **内容验证**：检查更新内容的准确性和完整性
- **一致性验证**：确保更新后的内容与系统其他部分保持一致
- **去重检查**：避免添加重复或相似的规则

**更新日志系统**：

- **详细记录**：记录所有更新操作的详细信息
- **时间戳**：每次更新都记录精确的时间戳
- **变更追踪**：追踪规则的变更轨迹，便于回溯和分析
- **性能监控**：记录更新耗时，监控系统性能

### 4. 钩子机制管理

管理和协调钩子系统的运行，实现功能的模块化和可扩展性：

**钩子检测机制**：

- **自动检测**：自动检测用户输入中的钩子格式（以`/`开头）
- **格式解析**：解析钩子名称和参数
- **精确匹配**：在order/SKILL.md中精确查找匹配的钩子定义

**钩子执行流程**：

- **参数验证**：验证钩子参数的有效性和完整性
- **技能加载**：加载钩子对应的技能实现
- **执行协调**：协调技能的执行过程
- **结果处理**：处理执行结果，生成响应

**钩子扩展支持**：

- **动态添加**：支持在对话过程中动态添加新钩子
- **自动记录**：自动记录未匹配的钩子，便于后续添加
- **格式规范**：确保新钩子符合格式规范
